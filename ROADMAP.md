# GitHunter 路线图 (ROADMAP)

---

## v1.0 — 重构版（当前）

> 目标：从单 HTML 文件升级为 TypeScript + React + Vite 可维护项目

**预计时间：** 1-2 周
**状态:** 🚧 进行中

### 完成项

- [x] Vite + React + TypeScript + Tailwind 工程化
- [x] 目录结构重设计（pages / components / services / hooks / types）
- [x] TypeScript 类型完整定义
- [x] GitHub API 服务层（含缓存 + 降级）
- [x] Gemini 服务层（含 5次重试 + 启发式降级）
- [x] 组件拆分（UserCard / Charts / Resume / Loading / Analysis）
- [x] SPEC.md / ROADMAP.md 文档

### 待完成

- [ ] 功能验证（`npm install && npm run dev`）
- [ ] CI/CD（GitHub Actions → GitHub Pages）
- [ ] 暗黑模式
- [ ] 完善 README

## v1.1 — 批量分析与团队对比

> 猎头最常见的场景：一次看 5-10 个候选人

**预计时间：** 2 周

### 任务

- [ ] 批量搜索模式（换行分隔多个用户名，一次分析多个）
- [ ] 团队对比视图（最多 4 人并排，雷达图叠加）
- [ ] 导出对比报告（CSV / PDF）
- [ ] 搜索历史（最近 20 条，本地保存）
- [ ] 收藏候选人（localStorage 保存精选列表）

---

## v1.2 — 真实 AI 增强

> 把启发式规则替换成真实 LLM 分析

**预计时间：** 1-2 周

### 任务

- [ ] DeepSeek / OpenAI API 接入（比 Gemini 便宜 10x）
- [ ] 基于仓库 README 的深度内容分析
- [ ] AI 为无描述仓库自动生成描述
- [ ] AI 生成面试问题建议
- [ ] 费用控制（每次分析最多消耗 $0.01）

---

## v1.3 — 简历增强与导出

> 做一份真正专业的一页纸简历

**预计时间：** 2 周

### 任务

- [ ] 精美简历模板（非当前简单排版）
- [ ] 多套简历模板可选
- [ ] 自定义简历内容（删除/重排 sections）
- [ ] HTML 简历直接生成 PDF（html2canvas + jsPDF）
- [ ] 候选人报告链接（生成唯一分享 URL）

---

## v1.4 — 组织/公司维度分析

> 从个人 → 团队/公司

**预计时间：** 2-3 周

### 任务

- [ ] 输入 GitHub Organization，获取所有成员列表

- [ ] 导出团队分析报告

## v2.0 — SaaS 平台化

> 从开源工具到付费 SaaS
